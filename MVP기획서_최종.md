# 돈이와 함께한 모든 순간들 MVP 기획서 (최종)

## 1. 문서 목적
- 본 문서는 현재 프로토타입(`index.html`)을 기준으로, 실제 구현/검수 가능한 MVP 요구사항을 확정하기 위한 실행용 PRD다.

## 2. 제품 정의
- 감정 태깅 기반 소비 기록을 통해 지출을 죄책감이 아닌 "나를 위한 시간 기록"으로 전환하는 모바일 중심 재무 다이어리.
- 핵심 메시지: "단순한 소비를 넘어, 가치 있는 순간을 기록하세요." / "Don't Worry, Be Happy"

## 3. MVP 목표
- 사용자가 월별 예산(수입/고정비)을 설정하고, 지출을 감정 태그와 함께 기록/회고할 수 있어야 한다.
- 사용자가 월 단위로 잔액을 확인하고, 자신의 소비 성향(성장/행복/생활/방황)을 시각적으로 이해할 수 있어야 한다.
- 기록 행위의 심리적 진입장벽을 낮추는 캐릭터 기반 인터랙션을 제공해야 한다.

## 4. 타깃 사용자
- 숫자 중심 가계부가 부담스러운 사용자
- 감정/라이프로그 방식으로 소비를 기록하고 싶은 사용자
- 월 단위로 예산과 실제 소비를 함께 관리하려는 1인 사용자

## 5. 범위 정의
### 5.1 In Scope (MVP)
- 월 이동(이전/다음) 기반 기록/설정/통계 조회
- 월별 설정: 수입 상세, 고정비 상세, 목표 입력
- 지출 기록: 생성/수정/삭제
- 지출 메타데이터: 날짜, 항목명, 금액, 태그, 선택 해시태그
- 잔액 계산 및 표시
- 통계: 태그별 비중(%), 월 메시지, 월 상세 히스토리
- 전월 설정 복사 UX(제안 + 적용)
- 로컬 저장(`localStorage`) 영속화

### 5.2 Out of Scope (MVP 제외)
- 계정/로그인/클라우드 동기화
- 다중 사용자 및 권한 체계
- 서버 백엔드/원격 DB
- 푸시 알림/리마인더
- 고급 분석(누적 추세, 목표 달성률 엔진, 추천)
- 금융기관 연동

## 6. 플랫폼/릴리즈
- 플랫폼: 모바일 우선 웹앱(반응형, 단일 페이지)
- 릴리즈 단계:
1. Alpha: 개인/내부 검증
2. Beta: 제한 사용자 검증
3. Public: 안정화 후 공개
- 릴리즈 목표일: `TBD`

## 7. 핵심 사용자 플로우
1. 앱 진입 -> 인트로에서 "기록하러 가기"
2. 홈에서 현재 월 잔액/약속 확인
3. 설정에서 월 수입/고정비/목표 입력
4. 채팅에서 지출 기록 추가(태그 포함)
5. 메모리에서 월 소비 성향 회고
6. 다음 달 이동 시 전월 설정 복사 여부 결정

## 8. 화면별 기능 요구사항
### 8.1 HOME
- 월 이동 가능
- 캐릭터 응원 문구 탭 시 랜덤 변경
- Current Balance 표시
- "이번 달 약속" 미입력 시 CTA, 입력 시 목표 텍스트 노출

### 8.2 CHAT
- 기본 시스템 메시지와 캐릭터 메시지 노출
- 지출 기록 생성/수정/삭제
- 각 기록 이후 태그 기반 캐릭터 반응 제공
- 신규 월 + 현재 설정 비어 있음 + 전월 설정 존재 시 복사 제안

### 8.3 MEMORY
- 태그별 소비 비율(%) 표시
- 최대 지출 태그 기반 월 코멘트 노출
- 상세 지출 내역 펼침/접기
- 월 차트 표시(현행: 해당 월 값 중심)

### 8.4 SETTING
- 수입 상세 항목 CRUD(추가/삭제)
- 고정비 상세 항목 CRUD(추가/삭제)
- 가용금액(수입-고정비) 표시
- 월 목표 저장

## 9. 데이터 모델 및 저장 정책
### 9.1 저장소
- 브라우저 `localStorage`
- 키:
  - `doni_test_exp_v2`
  - `doni_test_set_v2`

### 9.2 엔티티
- Expense:
  - `id`, `year`, `month`, `fullDate`, `item`, `cost`, `tag`, `customHash`
- MonthlySetting:
  - `inc`, `fix`, `incDetails[]`, `fixDetails[]`, `goal`

### 9.3 계산식
- 월 총지출(`spent`) = 해당 월 Expense `cost` 합
- 월 가용금액(`available`) = `incTotal - fixTotal`
- 홈 잔액(`balance`) = `available - spent`

## 10. 정책 정의
### 10.1 입력/검증 정책
- 지출 저장 필수값: 날짜, 항목명, 금액(0 초과), 태그
- 금액 입력은 천 단위 포맷 허용, 내부 저장은 숫자
- 해시태그는 입력 시 `#` 미포함이면 자동 부여

### 10.2 수정/삭제 정책
- 지출은 생성 월/과거 월 구분 없이 수정/삭제 가능(현행)
- 삭제는 즉시 반영(하드 삭제)

### 10.3 전월 복사 정책
- 복사 대상: 수입 총액/상세, 고정비 총액/상세
- 복사 제외: 목표(goal)는 빈 값으로 초기화

### 10.4 통화/시간 정책
- 통화: KRW 기준 정수 금액
- 날짜: 사용자 로컬 브라우저 기준

## 11. 톤앤매너 가이드
- 언어 톤: 위로, 동행, 비판 최소화
- 금지 방향: 죄책감 유발, 처벌형 카피
- 태그 의미:
  - 성장: 미래가치/자기계발
  - 행복: 즐거움/정서적 만족
  - 생활: 생활 유지/필수 소비
  - 방황: 비의도적/후회 가능 소비

## 12. 비기능 요구사항 (MVP)
- 성능: 월 기록 500건 기준 화면 전환/재렌더 1초 내 목표
- 신뢰성: 저장 실패 시 사용자 알림 제공(현행 일부 알림 있음, 보강 필요)
- 접근성: 주요 인터랙션 요소 터치 가능 크기 확보, 이미지 대체 텍스트 유지
- 보안/개인정보: 서버 전송 없음(로컬 저장), 단말 접근 리스크는 고지 필요

## 13. QA 수용 기준 (DoD)
- 기능:
1. 월 이동 시 HOME/CHAT/MEMORY/SETTING 날짜가 동기화된다.
2. 설정 입력 후 가용금액과 홈 잔액 계산이 일치한다.
3. 지출 생성/수정/삭제가 즉시 반영되고 재실행 후 유지된다.
4. 태그별 통계 비율이 월 지출 데이터와 일치한다.
5. 전월 복사 기능이 조건에 맞게 노출/적용된다.
- 안정성:
1. 새로고침 후 데이터 보존
2. 필수값 누락 시 저장 차단

## 14. KPI (MVP 관찰 지표)
- 첫 기록 완료율: 진입 사용자 중 첫 지출 입력 비율
- 주간 재기록률: 첫 기록 후 7일 내 재기록 비율
- 월 설정 완료율: 월 수입/고정비 중 1개 이상 등록 비율
- 오류율: 저장/렌더 오류 발생 비율

## 15. 리스크 및 후속 과제
- 로컬 저장 한계(기기 변경/브라우저 정리 시 데이터 손실)
- 통계 정교성 한계(연/누적 관점 미흡)
- 후속 우선순위:
1. 계정/동기화 + 백업/복원
2. 통계 고도화(월별 추세/목표 달성률)
3. 리마인더 및 루틴 설계
4. 캐릭터 반응 개인화
